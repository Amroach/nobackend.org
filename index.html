noBackend aims to kick off a discussion on how frontend developers wish to control backend tasks in a way that feels native. Our goal is to provide an extensive set of what we call Dreamcode to help providers of noBackend solutions get a better understanding of what their target group is looking for.

A beautiful API is hard to craft, but is the root of the success of libraries like jQuery that changed the way we build apps.

Dreamcode

Forget all the constraints of today's browsers. Then write down the code of your dreams for all the tasks you need to build your app. This is dreamcode. Here's another example: 

// share a report
var share = store.find("report", "id123").share();
share.grantWriteAccess(["jane@exam.pl", "joe@exam.pl"]);
share.grantReadAccess("boss@exam.pl");
share.ready( showSecretUrl );
share.fail( handleError );

Dreamcode is an excellent tool to drive the design process of building an app from the frontend's perspective. Instead of getting a RESTful API from the backend that you have to adapt to, all the backend should be hidden behind well thought through, reusable JavaScript APIs.

Dream code, discuss with us.

Example App

Inspired by the wonderful TodoMVC, we created an example invoicing app that is implemented using today's noBackend solutions.

It's based on Chris Coyier's Editable Invoice, that he created back in 2009 as a showcase for an HTML-only app. We've tweaked the design a bit, and also added features that typically require a backend: User Accounts, Exports, Emails.

See a list of noBackend example apps.
